# Kazakhstan NGO Recommended Pilot Plan

This plan turns the current discovery notes into an executable bounded pilot recommendation.

## Recommended Pilot Objective

Demonstrate that GrantFlow can reduce review chaos and shorten the path to a reviewable donor-aligned draft package for a Kazakhstan NGO team working across UN, EU, and USAID workflows.

## Phase 1 Scope

Recommended duration:
- `2-3` weeks

Recommended case count:
- `3` live or recent proposal cases

Recommended donor mix:
1. `un_agencies`
2. `eu`
3. `usaid`

Reason:
- This matches the customer's stated priority order.
- It gives one multilateral / UN-style case, one EU-style compliance case, and one MEL-heavy USAID case.

## Suggested Team

Minimum viable pilot team:

- Pilot owner: Head of Programs / Proposal Lead
- Operator: Grant / Proposal Manager
- Reviewer 1: MEL Specialist
- Reviewer 2: Technical Lead
- Approver: Country Director / Executive Director
- Ingest owner: Program Officer or MEAL Analyst

## Day-1 Decisions

The pilot should not start until these are explicit:

1. Which `3` cases are in scope
2. Who fills measured baseline data
3. Who uploads donor/source documents
4. Who is final approver for checkpoint release
5. Which metrics define pilot success

## Measured Baseline To Capture

For each selected case capture:

- time to first reviewable draft
- time to final review-ready package
- review loops
- number of active reviewers
- percentage of duplicate/conflicting comments if observable

Prefer customer data over estimates.

If exact data is unavailable, record:
- source
- confidence
- owner
- capture date

## Success Criteria For This Account

Recommend using the customer's own thresholds:

- `25-30%` reduction in time to first reviewable draft
- `1-2` fewer full review cycles
- lower duplicate/conflicting comment burden
- stronger LogFrame / MEL alignment with fewer reworks
- cleaner export package with less late-stage manual assembly

## What To Show In The Pilot

For each case:

1. Generate structured draft package
2. Show ToC + LogFrame + MEL outputs
3. Show critic findings and queue state
4. Show reviewer comments / actions
5. Export `.docx` / `.xlsx`
6. Compare against measured baseline

## What To Watch Closely

Operational risks for this account:

- too many reviewers too early
- unclear final approver
- donor corpus not loaded before live run
- pilot judged on final polish instead of workflow improvement

Content risks:

- UN agencies case quality must stay strong
- EU LogFrame / MEL framing must be credible for reviewer use
- USAID MEL / learning package must remain grounded and traceable

## Product Hypothesis To Validate

For this account, the main hypothesis is:

GrantFlow can reduce review coordination overhead and increase draft control faster than the team can achieve with manual document circulation plus generic AI tools.

## Recommended Output After Pilot

At pilot close, prepare:

- measured `pilot-metrics`
- `pilot-scorecard`
- `pilot-evidence-pack`
- short customer-specific lessons learned memo

## Commercial Read

If this pilot succeeds, the strongest commercial story is:

- proposal operations control
- review and approval workflow compression
- results-framework alignment
- auditability and export readiness

This is a stronger narrative than "faster text generation."
